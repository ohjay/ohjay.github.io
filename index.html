A full record (not for the faint of heart)

Due to the possibility of plagiarism, I'm unable to post my code for these projects. However, if you would like further information about any of them, I would be happy to answer your questions!

  • CS 61A | Python: Hog
    • A "greedy" dice-rolling game implemented in Python.
  • CS 61A | Python: Hog Contest
    • This was a contest that involved writing an optimized algorithm to play Hog. I entered and placed in the top 20 of a class with over 1000 students. (To be honest, I can't remember my exact placing, but I know for sure it was in the top 20.)
  • CS 61A | Python: Trends
    • A program to model and analyze the sentiment of USA tweets.
  • CS 61A | Python: Ants vs SomeBees
    • Designed to teach object-oriented principles, this project was modeled after Plants vs Zombies and was written in Python.
  • CS 61A | Python + Scheme: Scheme Interpreter
    • For the last full 61A project, we implemented a Scheme interpreter in Python. This interpreter could be run interactively, during which it would cycle repeatedly through the read-eval-print loop.
  • CS 61A | Scheme: Recursive Art Contest
    • Using Turtle graphics in Scheme, we were challenged to "unleash our inner recursive artist" by creating a visualization of some computational process. This was my entry; it was effectively a fusion of multiple terdragons.
  • CS 61B | Java: Checkers61b
    • A Java implementation of checkers, which was set apart from regular checkers by the inclusion of bomb and shield pieces.
  • CS 61B | Java: NGordNet
    • In this program, we investigated the history of words. It involved parsing Princeton's WordNet files and creating specific objects to visualize and analyze patterns in the data.
  • CS 61B | Java: Gitlet
    • Here we implemented (from the ground up!) our own version control system, which was modeled after Git.
  • CS 61B | Java: Fun with Tries
    • For the final project of 61B, we designed and wrote our own Trie classes and utilized them for sorting and autocompletion.
  • CS 61C | C + MIPS: MIPS Assembler/Linker [Part 1][Part 2]
    • An assembler written in C, alongside a linker written in MIPS.
  • CS 61C | MIPS + Logisim: Processor Design
    • A working processor built by wiring together logic gates. We had to implement all of the CPU logic, including the ALU and register file components.
  • CS 61C | Python: Performance Optimization/Convolutional Neural Networks
    • This project was about optimizing a CNN implementation such that it could identify at least 3000 cat images per second. We did this using methods along the lines of loop unrolling, SIMD instructions, and OpenMP.
  • CS 61C | Python: Deep Learning Algorithm + Spark [Part 1][Part 2]
    • Here, we again optimized a convolutional neural network – except this time we did it with Apache Spark and MapReduce. After we were finished, we ran our implementations on clusters of AWS EC2 servers in order to test our image-identifying code on a massive number of samples.

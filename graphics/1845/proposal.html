We intend to create an interactive scene that the user can virtually "walk" through, set within the MPI Informatics building in Saarbrücken, Germany. Background

As has already been mentioned, real-time rendering (RTR) is the main problem we want to solve. Formally, "real-time" is characterized by a framerate of 15 fps or faster. This serves as a challenge because normally for realistic, globally illuminated scenes we're forced to trace millions of rays around (and, by extension, check millions of intersections), just to render a single image. For our project, we want to render 15 of these images per second. When your name isn't Ren Ng, that's not easy.

We think that RTR is important because we view rendering speed as a critical measure of interactivity for many applications. For example, imagine watching a visualization of some car's test drive. If it ran in real-time, it could be really immersive – but if it froze before re-rendering every frame, it would quickly become frustrating.

To facilitate real-time rendering, we'll aim to implement sequential Monte Carlo instant radiosity alongside ray packets. To add real-time shadow effects, meanwhile, we'll look to follow some of the algorithms in Michael Schwarz's research paper.

In addition to the "real-time" thing, we'll strive to make our environments as photorealistic as we can over the next few weeks. This offers the potential for an endless number of extensions. For example, we might (especially if it makes rendering faster) adopt night as our time of day, and include perceptually-based tone mapping for low-light conditions. Alternatively, we could set our scene at twilight and add crepuscular rays coming in through a window. As a stretch (if we had nothing else to do, which is ludicrously unlikely), we could even add a fish tank, complete with some fluid simulation.

Incidentally, the scene is set in the Max Planck Institute for the simple reason that its building model is already available to us, complete with materials and light source data.
